STEP BY STEP

How to Build It — File Structure

1
Duplicate your .qmd pages Copy each page you want translated
2
Rename with language code index.qmd → index_jp.qmd
projects.qmd → projects_jp.qmd
3
Translate the content Titles, body text, chip labels — everything visible
4
Register in _quarto.yml Add JP pages to the render list
5
Add the language switcher include-after-body: lang-switch.html
# _quarto.yml — render list
project:
  render:
    - "*.qmd"  # all top-level
    - "blogs/*.qmd"
    - "projects/*.qmd"
    - "projects_jp/*.qmd"

format:
  html:
    include-after-body:
      lang-switch.html

# File structure:
├── index.qmd
├── index_jp.qmd
├── resume.qmd
├── resume_jp.qmd
├── projects.qmd
├── projects_jp.qmd
└── lang-switch.html
THE MAGIC FILE

lang-switch.html — How the Button Works

lang-switch.html
// Page mapping — EN ↔ JP
var EN_TO_JP = {
  'index.html': 'index_jp.html',
  'resume.html': 'resume_jp.html',
  'projects.html': 'projects_jp.html'
};

// Detect current page language
function detectPage(path) {
  for (enKey in EN_TO_JP) {
    if (path.endsWith(enKey))
      return { lang: 'en', ... };
  }
}

// Remember user's choice
localStorage.setItem('site-lang', lang);

// Inject dropdown into navbar
ul.innerHTML = '<li>🇺🇸 EN | 🇯🇵 JP</li>';
🗺️
Page Mapping A simple JS object connects each English page to its Japanese equivalent. You only add what you've translated.
🔍
Auto-Detection The script reads the current URL and figures out which language you're on — no manual setup per page.
💾
Memory via localStorage User's language choice is saved in the browser. If they pick JP, every page auto-redirects to the JP version.
🧩
Injects into Navbar The dropdown button appears in your top-right nav automatically — no edits to each .qmd file needed.
