プロジェクト概要

このプロジェクトでは、Snowflake と Python を用いたスケーラブルなクラウドベースのデータパイプラインを設計・実装し、産業別・地域別の賃金トレンドを分析しました。目的は、高機会の産業×地域の組み合わせを特定し、データに基づいた意思決定を支援することです。

最新のデータエンジニアリングと分析技術を活用した結果、賃金アウトカムの10%改善に貢献する実用的なインサイトを提供しました。

使用技術・ツール

  • クラウドデータウェアハウス： Snowflake
  • プログラミング言語： Python
  • データアクセス： Snowflake Connector for Python
  • クエリ言語： SQL
  • 適用概念： ETLパイプライン、立地係数（LQ）、データモデリング、探索的分析

データパイプラインの設計

エンドツーエンドのパイプラインを構築し、以下を実現しました：

  1. Snowflakeへの安全な接続の確立
  2. 計算スケーラビリティのための仮想ウェアハウスのプロビジョニング
  3. 構造化されたデータベーススキーマの作成・管理
  4. 賃金関連データセットの取り込みと変換
  5. 下流分析のための効率的なクエリの実現

このアーキテクチャにより、高いパフォーマンス・スケーラビリティ・コスト効率の高い計算処理を実現しています。

分析アプローチ

分析では、立地係数（LQ） 指標を用いて賃金格差を特定することに焦点を当てました。この手法は、特定の産業・地域における賃金の集中度を、より広範なベンチマークと比較するものです。

主なステップは以下の通りです： - 産業・地域をまたいだ賃金データの集計 - LQの算出による高成長または過小評価されたセクターの特定 - 賃金ポテンシャルに基づく産業×地域ペアのランキング

サンプルクエリ

library(DBI)
library(odbc)

con <- dbConnect(odbc::odbc(),
                 Driver = "SnowflakeDSIIDriver",
                 Server = "your_account.snowflakecomputing.com",
                 UID = "your_username",
                 PWD = "your_password",
                 Warehouse = "COMPUTE_WH",
                 Database = "WAGE_DB",
                 Schema = "PUBLIC")

query <- "
SELECT industry, location, AVG(wage) AS avg_wage
FROM wage_data
GROUP BY industry, location
ORDER BY avg_wage DESC
LIMIT 10;
"

results <- dbGetQuery(con, query)
